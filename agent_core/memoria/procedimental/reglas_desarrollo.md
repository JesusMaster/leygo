
- Como medida de seguridad y control de calidad, NUNCA se debe iniciar el desarrollo de una nueva funcionalidad o la creación de un subagente sin antes presentar al usuario, el plan de trabajo detallado y obtener su aprobación explícita.

### El flujo obligatorio es:
1. Análisis de la solicitud.
2. Presentación del plan/propuesta al usuario.
3. Espera de validación/aprobación.
4. Ejecución del desarrollo.
